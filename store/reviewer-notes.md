# Reviewer notes

1. Installing the extension replaces `chrome://newtab` with `dashboard.html`.
2. The three-step onboarding explains the New Tab override and local bookmark handling. Its compact inspiration-folder menu fixes Ampira’s packaged preset as the first option and lists personal bookmark folders below it, then onboarding requests optional exact website-origin and Chrome-provided website-icon access before opening the dashboard. Preset mode can finish with no top-level inspiration folder. Switching later preserves the personal folder and never writes bookmarks. The detailed AI data-transfer disclosure and affirmative consent control appear in Settings → AI before an AI key can be saved, tested, or used.
3. The required permissions are only `bookmarks`, `storage`, and `alarms`. The toolbar icon opens the packaged Ampira dashboard and does not read the current tab. Reading-queue records remain local unless the user separately enables the default-off Chrome Sync switch. The extension only calls bookmark read methods. The optional `favicon` permission is requested from onboarding or Settings → Browser and can be removed there; without it, Ampira uses its packaged fallback icon.
4. To test Feed fetching, grant the listed exact origins in the final onboarding step. Settings → News shows effective coverage and per-source diagnostics, including the extraction method and a single-source retry. If an allowed landing page declares a Feed on another domain, that exact domain appears as a separate user-gesture permission request and receives no request before approval. All origins remain individually manageable under Settings → Browser; declining leaves bookmark cards usable.
5. In preset mode, verify the 48 links use 24 packaged local covers, each cover appears on exactly two sites, daily batches do not repeat a cover, and the first three reshuffles contain 15 different sites and covers. Preset covers make no metadata, remote-image, or Brave requests and the 48 origins are absent from the permission list. In personal-bookmark mode, grant exact source origins under Settings → Browser and open a new tab. Personal inspiration cards read only inert public page metadata, try up to three original-page candidates, and start preloading all three reshuffle batches before first render. An authorized same-origin news article with no Feed image may receive one bounded metadata read; cross-origin article metadata reads are skipped. To test fallback, optionally enable Brave Image Search, grant `api.search.brave.com`, and use a non-production test key. A news or personal inspiration card with no usable original image, or with deliberately failed original candidates, may then use Brave; unrelated URLs, preset covers, and Reader images remain excluded. Revoking a source origin stops its metadata reads and removes the corresponding image cache while leaving the bookmark usable.
6. AI is optional. To test it, open Settings → AI, review and affirmatively accept the disclosure, grant the configured provider origin, enter a non-production test key, save, and run the connection test. The key is sent directly to that provider to authenticate the request and never to an Ampira backend. Remove the key afterward.
7. The first compact efficiency card cycles through Today's events → Weather forecast → To-do list with one persistent Switch button. To test Weather, switch once, enter `慈溪`, and approve only `https://geocoding-api.open-meteo.com/*` and `https://api.open-meteo.com/*`. The bundled GeoNames-derived China index returns `慈溪市 · 宁波市 · 浙江省 · 中国` without sending that search text to a geocoder; its attribution is visible below the candidates. A non-China city query falls back directly to Open-Meteo's geocoder. Ampira never auto-selects ambiguous candidates or uses geolocation. Only the selected coordinates go directly to Open-Meteo's forecast API, not to an Ampira backend. Settings → Browser lists both origins after opt-in; revoke either one and confirm requests stop, the forecast cache is removed, and the selected city remains available for reauthorization. Weather attribution is visible in the forecast card.
8. To test To-do, switch again, add an item with Return, complete, restore, and delete it. To-dos, the selected weather city, and reading queue remain local by default. In Settings → Browser → Cross-device content, each category has a separate default-off Chrome Sync switch. Enable one, confirm the corresponding content appears in another signed-in synced Chrome installation, then disable it and confirm the Chrome Sync copy is removed while the current local copy remains. The selected utility mode and forecast cache never sync.
9. The extension contains no content scripts, remote JavaScript, `eval`, tracking, advertising, payment flow, or developer backend.
10. The in-app reader fetches public HTML only after exact-origin permission, then renders inert structured text and images. Videos remain explicit source links; no remote iframe or player is embedded.
11. Click the Ampira toolbar icon and confirm it opens the packaged dashboard without requesting or reading the current tab. Add and remove a displayed news or bookmark card from Reading queue, then verify the default-local behavior and the optional Sync reading queue switch separately.
12. Incognito use is disabled. General web search is not provided; the top search field filters Ampira content only.
