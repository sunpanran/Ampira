Ampira 隐私政策

生效日期：2026-07-11

Ampira 的单一用途是把用户选择的 Chrome 书签和资讯来源整理成可搜索的新标签页信息看板。Ampira 没有开发者服务器、账号系统、广告或分析。

处理的数据

  • 用户选择的书签标题、网址与文件夹结构。
  • 用户授权来源的公开 Feed、网页标题、摘要、正文和图片地址。
  • 界面设置、阅读队列、归档、来源偏好、缓存和搜索问题。
  • 用户自愿配置的 AI 或图片搜索 API 密钥。

用途和存储

Ampira 使用 chrome.storage.sync 保存部分非凭据设置；如果用户开启 Chrome Sync，Chrome 可能把这些设置复制到该账号登录的其他 Chrome 安装。资讯、摘要、搜索缓存和阅读状态保存在扩展 IndexedDB。API 密钥只保存在当前 Chrome 配置的 chrome.storage.local 中，不参与 Chrome Sync，保存后页面不会回显完整密钥。

公共 Feed 补盲默认开启，用于在书签来源之外补充公开资讯，用户可以在设置中关闭。Ampira 只有在用户授权相应精确来源域名后才会请求这些 Feed。

第三方传输

未经用户操作，Ampira 不向开发者或第三方发送书签、阅读状态或搜索问题。用户授权来源后，扩展会直接连接对应公开网站。界面卡片或阅读器显示远程图片时，浏览器会直接请求图片所在网站；即使不发送来源页信息，该网站仍会收到请求必需的常规网络信息，例如 IP 地址和浏览器信息。Ampira 不代理这些请求。

用户明确启用 AI 后，提交的问题、文章标题、选定正文以及提供上下文所需的文章网址会发送给用户配置的 AI 服务商。根据功能路径和扩展版本，文章网址可能是来源中保存的网址，也可能是移除不必要信息后的规范化或最小化网址；网址仍可能包含路径或查询信息，应将其视为已传输数据。AI API 密钥会随请求直接发送给用户配置的服务商，仅用于该服务商验证请求，不会发送给 Ampira 开发者。

用户启用 Brave 图片搜索后，生成的图片查询和对应 API 密钥会直接发送给 Brave Search API，密钥用于验证请求。以上第三方传输均使用 HTTPS；HTTP 仅允许连接用户配置的本机 localhost 服务。

权限

bookmarks 用于只读整理书签；Ampira 不调用书签写入接口。storage 用于本地设置和缓存。alarms 用于每 15 分钟检查到期来源。网站权限为可选权限，只在用户点击授权时按具体域名申请。

共享、出售与保留

Ampira 不出售、共享或将数据用于广告定位。资讯缓存最多保留 30 天并限制为约 25MB；用户可以随时清除缓存。卸载扩展会清除扩展本地数据，Chrome Sync 中的设置按 Chrome 的同步规则处理。

Chrome Web Store Limited Use

Ampira 对从 Google API 获取的信息的使用将遵守 Chrome Web Store User Data Policy，包括 Limited Use 要求。Ampira 只为已披露的单一用途和面向用户的功能使用这些数据，不将其用于个性化广告、信用评估或数据转售。Ampira 没有可供开发者或其人员读取这些数据的后端。

联系

隐私或支持问题请使用项目支持入口；请勿附上 API 密钥、私人书签或个人浏览信息。
