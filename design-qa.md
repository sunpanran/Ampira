# Ampira MV3 QA

本文件只描述当前 Manifest V3 Chrome 扩展的验证状态。已移除的 Node 版实现不属于当前运行架构；旧 `dashboard-cache/` 仅是用户自行管理的历史数据，本项目不会读取、迁移、删除或打包它。

## Current Architecture

- `dashboard.html` 是新标签页单页入口；浏览器代码使用本地 ESM，不加载远程脚本或执行远程代码。
- `extension/service-worker.mjs` 注册 Chrome 事件和消息入口；设置归一化、有界网络读取、配额和客户端状态写入位于 `extension/core/` 的可测试模块。
- 必需权限保持为 `bookmarks`、`storage`、`alarms`；网站访问只通过用户手势申请精确可选来源。
- 部分非凭据设置位于 Chrome Sync；本机凭据位于 `chrome.storage.local`，Feed、Reader、摘要和阅读状态位于 IndexedDB。
- UI 支持简体中文、繁体中文和英文；远程文本仅通过结构化 DOM 与 `textContent` 渲染。

## Automated Verification

每次功能修改必须运行：

```powershell
.\scripts\verify-extension.ps1
```

当前测试覆盖：

- Manifest 权限、CSP、三语键一致性和远程代码禁令。
- 根页面资源与 `docs/` 全部 HTML 子页的本地资源和链接完整性。
- 书签角色去重、嵌套分类上限、来源排除和精确域名生成。
- RSS、Atom、JSON Feed、XML 实体 URL、来源安全上限、网络失败和去重排序。
- 设置白名单与边界归一化、本机密钥隔离、缓存裁剪和旧凭据清理。
- 大型设置的 Chrome Sync 分块与并发写入串行化、客户端状态串行合并与批量发送、非法状态载荷拒绝、并发 AI 配额预留。
- Reader 主体提取、安全链接、图片/视频结构、缓存回退、HTTP 分类、超时语义及无长度响应的流式大小限制。
- Brave 图片预览的权限、大小限制、安全搜索、低价值图片过滤、并发请求去重和缓存降级路径。
- 刷新启动锁、强制刷新排队、缓存清理代际失效，以及失败来源保留最后成功内容、有效空来源清理旧内容。
- 设置草稿差异、取消恢复、未知字段隔离，以及优先级、去重、未读筛选和分页等前端纯选择器。

## Browser Verification

UI 或交互修改后，在安全本地预览或加载已解压扩展中验证：

- 视口：`1280×800`、`1440×1000`、`390×844`。
- 文档、设置弹窗、引导页和 Reader 横向溢出均为 `0`。
- 控制台无错误；刷新、搜索、换批、右键菜单、稍后读、已读和跨重载持久化正常。
- 权限只能由用户操作触发；拒绝权限后页面保持可用，非本地 HTTP 不进入 Reader。
- 深色、浅色、系统模式及三种语言均保持现有信息终端布局和断点。
- 有意保留的截图写入 `output/playwright/`。

## Release Verification

```powershell
.\scripts\verify-extension.ps1 -Package
```

打包脚本从 `manifest.json` 读取版本，使用允许列表复制文件，并在压缩前拒绝以下内容：

- `dashboard-cache/`、测试、输出、隐藏配置或非允许列表文件。
- 疑似 API 密钥、本机绝对路径和远程脚本。
- 非 Manifest V3 清单或与显式版本参数不一致的版本。

发布脚本要求 PowerShell 7 与 Node.js 20+。打包模式会拒绝未替换占位和未配置或不匹配的 `REQUIRED_SUPPORT_URL`。它在覆盖已有版本化 ZIP 前完成环境和输入检查，按稳定路径顺序与固定时间戳创建 ZIP，并同时生成 `.zip.sha256` 与包含逐文件 SHA-256、可选 `GITHUB_SHA` 的 `.manifest.json`。Windows CI 固定使用 Node.js 20 和 `pwsh` 运行验证/打包并上传这三项产物；没有真实支持 URL 时按设计阻止发布。

上传前仍需人工确认 ZIP 根目录包含 `manifest.json`，隐私/支持页面已发布，商店文案、权限说明和实际行为一致。

- Brave 图片兜底从占位路由改为真实的有界搜索、筛选、负缓存与客户端失败隔离。

## 2026-07-11 Release Hardening

- OpenAI 密钥、服务地址、调用方式、模型与凭据代际改为同一条本机原子 Provider 记录；旧本机密钥安全迁移，Brave 密钥继续独立保存在扩展本地。
- 书签同意、首次引导和 AI 数据传输同意改为设备级本机状态；同意必须匹配当前版本，AI 同意还必须匹配当前 Provider origin。
- Provider origin 改变时界面会撤销旧勾选并要求重新确认；测试连接、正常 AI 调用和缓存身份均使用当前本机 Provider 代际。
- Feed 与 Reader 禁止自动重定向；Feed 显式省略凭据和 Referer。权限列表会区分当前需要、宽泛覆盖与遗留 origin，并检查权限 API 的真实布尔结果。缓存提交已串行化且带失效代际；撤权会按 Feed、AI、Brave、Reader 能力定向清理，Feed、摘要、问答与预览还会在读取时按当前权限 fail-closed 复核。
- 三语引导、隐私政策、商店隐私字段和审核说明补齐公共 Feed、Chrome Sync、AI URL/API key、远程图片请求与 Limited Use 披露。
- 发布版本提升至 `26.1.4`；PowerShell 7/Node.js 20 发布门禁、确定性 ZIP、SHA-256/逐文件清单侧车和 Windows CI 已建立。真实支持入口固定为 `https://github.com/sunpanran/Ampira/issues`，当前版本发布包仍需通过同值环境变量与完整发布门禁生成。
